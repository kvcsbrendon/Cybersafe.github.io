Software and Hardware Security Measures

Understanding Software and Hardware Security Measures

  • Antivirus Software and Detection Techniques
    • Virus Signatures: Identifying malware by comparing files to a database of known virus signatures.
    • Heuristics: Techniques to identify potentially suspicious file content based on behavior analysis.
    • Dealing with Threats: Methods for handling identified threats, such as quarantine and removal.
  • Software and Hardware Firewalls
    • Packet Filtering and Inspection: Filtering network traffic based on predefined rules.
    • Application Layer Awareness: Inspecting traffic at the application layer to detect threats.
    • Inbound and Outbound Rules: Defining rules to control the flow of incoming and outgoing traffic.
    • Network Address Translation (NAT): Converting private IP addresses to a public IP address for secure communication.
  • User Authentication
    • User Login Procedures: Processes for verifying a user's identity.
    • Strong Passwords: Use of complex passwords to enhance security.
    • Text and Graphical Passwords: Different methods of authentication using text or images.
    • Biometric Authentication: Using physical characteristics like fingerprints or retina scans.
    • Two-Step Verification: Additional verification step beyond the password.
    • Security Tokens: Physical devices such as USB-based and near-field keys used for authentication.
    • Knowledge-Based Authentication: Using question and response pairs to verify identity.
    • Kerberos: Network authentication protocol for secure user authentication.
    • Certificate-Based Authentication: Using digital certificates to authenticate users.
  • Access Controls
    • Resource Restrictions: Methods to restrict user access to applications, folders, files, and physical resources.
  • Trusted Computing
    • Trusted Computing: Ensuring systems are secure from the hardware level up.
  • Purpose and Uses of Encryption
    • Safe Password Storage: Encrypting passwords to protect them from unauthorized access.
    • Digital Rights Management (DRM): Controlling access to digital content.
    • File, Folder, Disc Encryption: Encrypting data to prevent unauthorized access.
    • Communications Encryption: Encrypting data in transit to protect against interception.
    • Built-In Device Encryption: Encryption features in smartphones and tablets.
    • The Onion Router (Tor): Anonymous communication through encrypted routing.
    • Virtual Private Networks (VPNs): Secure, encrypted connections over public networks.
    • Digital Certificates and Certificate Authorities: Ensuring secure communications through verified certificates.
    • Hypertext Transfer Protocol Secure (HTTPS): Secure web browsing with encrypted communication.
    • Public/Private Keys: Asymmetric encryption for secure communication and data protection.
  • Protecting Wireless LANs
    • MAC Address Filtering: Restricting network access to specific device MAC addresses.
    • Hiding SSID: Preventing the network name from being broadcasted.
    • Wireless Encryption: Using encryption protocols like WEP, WPA2, and WPS to secure wireless networks.
    • Mitigating Wireless Vulnerabilities: Addressing known issues and implementing security best practices.
    • Security in Network Design: Ensuring security considerations are integrated from the development stage.
